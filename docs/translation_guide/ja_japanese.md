## 1. トーンとレジスター

### Synoptic Studio 向け

- **レジスター**：丁寧語（です・ます体）
- **トーン**：信頼感があり、明確で、親切 — 決して偉そうにしない
- **文体**：能動態が基本；受動態はアクションを強調する時のみ

### 基本原則

```
✅ 「プロジェクトを保存してください」（直接的、能動的）
❌ 「プロジェクトは保存されます」（受動的、距離感がある）

✅ 「美しいバイリンガル本を作成しましょう」
❌ 「バイリンガル本はあなたによって作成できます」
```

## 2. 敬語レベル

### Synoptic のルール

**丁寧語（です・ます体）を使用** — ビジネスアプリケーションの標準です。

| 場面             | スタイル              | 例                         |
| ---------------- | --------------------- | -------------------------- |
| UI 全般          | **丁寧語**            | 「保存してください」       |
| ボタン           | **敬体または常体**    | 「保存」「保存する」       |
| エラーメッセージ | **丁寧語**            | 「接続を確認してください」 |
| マーケティング   | **丁寧語/普通体混合** | 「今すぐ始めよう」         |

### 敬語の種類と使い分け

| 種類       | 説明               | UI での使用   |
| ---------- | ------------------ | ------------- |
| **尊敬語** | 相手の動作を高める | 通常不要      |
| **謙譲語** | 自分の動作を低める | 通常不要      |
| **丁寧語** | 文末を丁寧にする   | ✅ これを使用 |

### ⚠️ 注意点

- 「お」「ご」を付けて丁寧さを表現：「お名前」「ご登録」「お知らせ」
- 過度な敬語は逆効果 — 自然な丁寧さを心がける

## 3. ジェンダーニュートラルな言葉遣い

### 日本語の特徴

日本語の名詞には文法的な性別がありません。これは自然にジェンダーニュートラルです。

### 代名詞の使用

| 代名詞   | 性別 | 推奨          |
| -------- | ---- | ------------- |
| 彼       | 男性 | UI では避ける |
| 彼女     | 女性 | UI では避ける |
| その方   | 中性 | ✅ 使用可     |
| （省略） | —    | ✅ 最も自然   |

### ベストプラクティス

```
✅ 「ユーザーはプロジェクトを保存できます」（主語明示）
✅ 「プロジェクトを保存できます」（主語省略 — 最も自然）
❌ 「彼らのプロジェクト」

✅ 「著者と編集者の皆様」
✅ 「クリエイターの皆様」
```

## 4. 翻訳の落とし穴とよくある間違い

### 偽の友（False Friends）

| 英語      | ❌ 誤訳      | ✅ 正しい訳             | 説明                         |
| --------- | ------------ | ----------------------- | ---------------------------- |
| Save      | セーブ       | 保存                    | 「セーブ」はゲーム用語       |
| Delete    | デリート     | 削除                    | 日本語を優先                 |
| Library   | ライブラリー | ライブラリ/ライブラリー | 長音符の有無                 |
| Character | キャラクター | 文字                    | ソフトウェア文脈では「文字」 |
| Execute   | 処刑する     | 実行する                | 文脈による                   |
| Resume    | 履歴書       | 再開する                | 文脈による                   |

### 「Ultimate」の翻訳

英語の「ultimate」は**「最高の、究極の」**を意味します。

```
✅ 「究極のバイリンガルスタジオ」— 正しい意味
✅ 「最高のバイリンガルスタジオ」— より自然
❌ 「最後のバイリンガルスタジオ」— 「終わり」を暗示
```

## 5. 文法と構文

### 文の構造

日本語は**SOV（主語-目的語-動詞）**構造：

```
英語: I save the project (SVO)
日本語: 私は プロジェクトを 保存する (SOV)
```

### 主語の省略

日本語では主語を省略することが自然で一般的：

```
✅ 「プロジェクトを保存しました」（主語省略 — 自然）
△ 「あなたのプロジェクトを保存しました」（主語明示 — やや冗長）
```

### 助詞の重要性

| 助詞 | 機能             | 例                                   |
| ---- | ---------------- | ------------------------------------ |
| は   | トピックマーカー | 「プロジェクト**は**保存されました」 |
| が   | 主語マーカー     | 「エラー**が**発生しました」         |
| を   | 目的語マーカー   | 「ファイル**を**削除する」           |
| に   | 方向/対象        | 「PDF**に**エクスポートする」        |
| で   | 手段/場所        | 「クラウド**で**同期する」           |
| の   | 所有/修飾        | 「あなた**の**プロジェクト」         |

## 6. 文字の使い分け

### 3 つの文字体系

| 文字         | 用途                   | 例                     |
| ------------ | ---------------------- | ---------------------- |
| **ひらがな** | 文法要素、やまとことば | する、できます         |
| **カタカナ** | 外来語、強調、技術用語 | ファイル、プロジェクト |
| **漢字**     | 名詞、動詞語幹、形容詞 | 保存、削除、設定       |

### カタカナ語の注意点

- 長音符「ー」の使用：「コンピューター」vs「コンピュータ」
- 業界標準に従う：IT 業界では長音符を省略する傾向

## 10. 文化的配慮

### デザイン文化

日本のユーザーは：

- 情報密度の高い UI に慣れている
- 丁寧な表現を期待する
- 細部への配慮を高く評価する

### 色の象徴

- 赤：警告、エラー、重要
- 緑：成功、完了
- 白と黒：洗練、モダン

### 数字の文化的意味

- **4**：「死」と同音 — 価格設定で避けることも
- **9**：「苦」と同音 — 同様に注意
