# 中文简体 (zh) — 翻译指南

## 概述

| 属性         | 值                                 |
| ------------ | ---------------------------------- |
| **ISO 代码** | `zh`                               |
| **本地名称** | 简体中文 / 普通话                  |
| **文字系统** | 简化汉字                           |
| **书写方向** | 从左到右 (LTR)                     |
| **主要地区** | 中国大陆、新加坡、马来西亚华人社区 |

---

## 1. 语气与语域

### Synoptic Studio 界面

- **语域**：专业但亲切
- **语气**：自信、清晰、有帮助——从不居高临下
- **语态**：主动语态为主；被动语态仅用于强调动作

### 关键原则

```
✅ 「保存您的项目」（直接、主动）
❌ 「您的项目将被保存」（被动）

✅ 「创建精美的双语图书」
❌ 「双语图书可以由您创建」
```

---

## 2. 称谓：你 vs. 您

### Synoptic 规则

**使用「您」** — 这是专业应用程序的标准。

| 场景       | 形式   | 示例                         |
| ---------- | ------ | ---------------------------- |
| 用户界面   | **您** | 「保存您的项目」             |
| 技术文档   | **您** | 「您可以导出为 PDF」         |
| 错误消息   | **您** | 「请检查您的网络连接」       |
| 非正式营销 | **你** | 「开启你的创作之旅」（可选） |

### 注意事项

- 「您」比「你」更正式、更尊重
- 在整个应用中保持一致
- 台湾和香港用户也期望使用「您」

---

## 3. 性别中立语言

### 中文的优势

中文天然具有性别中立性：

- 代词「他/她/它」书写不同，但口语中发音相同（tā）
- 许多名词没有语法性别

### 最佳实践

```
✅ 「用户可以保存项目」（无性别标记）
✅ 「作者和编辑」（中性）
❌ 「男作者」（除非特别需要）

✅ 「您的团队」
✅ 「创作者」
```

### 第三人称代词

- 书面语中，使用「他」作为通用代词仍然常见
- 或使用重复名词避免代词：「用户...用户...」
- 使用「其」作为正式的无性别代词

## 4. 翻译陷阱与注意事项

### 常见翻译错误

| 英文      | ❌ 错误翻译 | ✅ 正确翻译               |
| --------- | ----------- | ------------------------- |
| Ultimate  | 最后的      | 终极的、顶级的            |
| Save      | 拯救        | 保存                      |
| Character | 角色        | 字符 / 角色（根据上下文） |
| Library   | 图书馆      | 库（软件）/ 资料库        |
| Execute   | 处决        | 执行                      |
| Resume    | 简历        | 继续 / 简历（根据上下文） |

### 「Ultimate」的翻译

在英文中，「ultimate」通常表示**「最好的，终极的」**。
中文「终极」是合适的翻译。

```
✅ 「终极双语工作室」
✅ 「顶级双语创作平台」
❌ 「最后的双语工作室」（暗示结束）
```

## 5. 语法与句法

### 句子结构

- **主语-谓语-宾语 (SVO)** 是标准结构
- 中文没有动词变位或名词变格
- 时态通过上下文或时间词表达

### 量词

中文需要在数字和名词之间使用量词：

| 物品 | 量词  | 示例     |
| ---- | ----- | -------- |
| 书籍 | 本    | 一本书   |
| 文件 | 个/份 | 三个文件 |
| 页面 | 页    | 第五页   |
| 项目 | 个    | 两个项目 |

### 标点符号

使用中文标点符号（全角）：

| 符号   | 中文      | 英文   |
| ------ | --------- | ------ |
| 句号   | 。        | .      |
| 逗号   | ，        | ,      |
| 问号   | ？        | ?      |
| 感叹号 | ！        | !      |
| 冒号   | ：        | :      |
| 引号   | 「」或 "" | ""     |
| 书名号 | 《》      | 无对应 |

## 6. 排版规范

### 引号

- 直角引号首选：**「 」**
- 嵌套引号：**『 』**
- 或使用弯引号：**" "**

```
✅ 「您的项目『我的书』已保存」
✅ "您的项目'我的书'已保存"
```

### 数字格式

| 类型       | 格式                    | 示例                |
| ---------- | ----------------------- | ------------------- |
| 千位分隔符 | 逗号或无                | 1,000 或 1000       |
| 小数点     | 句点                    | 3.14                |
| 货币 (¥)   | ¥X.XX                   | ¥12.99              |
| 货币 (€)   | €X.XX                   | €12.99              |
| 百分比     | X%                      | 50%                 |
| 日期       | YYYY 年 MM 月 DD 日     | 2025 年 12 月 30 日 |
| 时间       | HH:MM 或 上午/下午 X:XX | 14:30 或 下午 2:30  |

### 中英文混排

在中文和英文/数字之间添加空格：

```
✅ 「导出为 PDF 格式」
✅ 「支持 30 多种语言」
❌ 「导出为PDF格式」
```

## 7. 界面模式

### 按钮

- 使用**动词**或**动词短语**
- 最多 1-4 个字符

| 操作     | 中文        |
| -------- | ----------- |
| Save     | 保存        |
| Export   | 导出        |
| Delete   | 删除        |
| Cancel   | 取消        |
| Create   | 新建 / 创建 |
| Edit     | 编辑        |
| Sign in  | 登录        |
| Sign out | 退出 / 登出 |

### 错误消息

- **具体**说明问题
- 提供**解决方案**
- 不要责怪用户

```
✅ 「网络连接已断开。请检查您的网络设置后重试。」
❌ 「网络错误」
❌ 「您断开了连接」
```

## 9. 地区差异

### 简体 vs. 繁体

| 简体 (大陆) | 繁体 (台湾/香港) |
| ----------- | ---------------- |
| 软件        | 軟體             |
| 信息        | 資訊             |
| 视频        | 視頻 / 影片      |
| 网络        | 網絡 / 網路      |
| 文件夹      | 資料夾           |

### Synoptic 建议

- 为简体中文 (zh) 和繁体中文 (zh-TW) 分别创建翻译
- 不要自动转换 — 术语和表达方式不同

---

## 10. 文化注意事项

### 本地化最佳实践

- 避免敏感的政治话题
- 注意颜色象征意义（红色 = 幸运，白色 = 哀悼）
- 日期格式使用中国标准

### 数字忌讳

- **4** 在中文中与「死」谐音 — 在定价或编号中避免
- **8** 是幸运数字
