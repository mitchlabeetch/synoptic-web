## 1. Ton & Registre

### Pour l'interface Synoptic Studio

- **Registre** : Professionnel mais accessible
- **Ton** : Confiant, clair, bienveillant — jamais condescendant
- **Voix** : Active de préférence, passive uniquement pour insister sur l'action

### Principes clés

```
✅ « Enregistrez votre projet » (direct, actif)
❌ « Votre projet sera enregistré » (passif)

✅ « Créez de magnifiques livres bilingues »
❌ « Des livres bilingues peuvent être créés par vous »
```

---

## 2. Tutoiement vs Vouvoiement

### Règle pour Synoptic

**Utilisez le vouvoiement (vous)** — c'est le standard pour les applications professionnelles.

| Contexte                | Forme           | Exemple                                |
| ----------------------- | --------------- | -------------------------------------- |
| Interface utilisateur   | **Vous**        | « Enregistrez votre projet »           |
| Documentation technique | **Vous**        | « Vous pouvez exporter en PDF »        |
| Messages d'erreur       | **Vous**        | « Vérifiez votre connexion »           |
| Marketing amical        | **Tu** possible | « Crée ton premier livre » (optionnel) |

### ⚠️ Attention aux accords

Le vouvoiement impose des accords singuliers quand on s'adresse à une personne :

```
✅ « Vous êtes connecté » (singulier, une personne)
❌ « Vous êtes connectés » (pluriel, plusieurs personnes)
```

## 3. Écriture Inclusive — L'écriture épicène

### Options recommandées

| Méthode                | Exemple                         | Usage                    |
| ---------------------- | ------------------------------- | ------------------------ |
| **Doublet abrégé**     | utilisateur·rice                | Recommandé pour titres   |
| **Doublet complet**    | utilisateur ou utilisatrice     | Textes longs             |
| **Formulation neutre** | « les personnes qui utilisent » | Préféré pour la fluidité |
| **Point médian**       | auteur·e·s                      | Compact mais contesté    |

### Bonnes pratiques

```
✅ « Bienvenue ! » (neutre)
❌ « Bienvenue, cher utilisateur » (masculin par défaut)

✅ « Les personnes abonnées »
❌ « Les abonnés » (masculin générique)

✅ « Votre équipe éditoriale »
❌ « Vos éditeurs »
```

### Points de vigilance

- Le **point médian (·)** est préféré au point classique : `auteur·rice` plutôt que `auteur.rice`
- Évitez les parenthèses : `auteur(rice)` est déconseillé
- Certains contextes officiels n'acceptent pas l'écriture inclusive — adaptez si nécessaire

## 4. Faux-amis & Pièges de Traduction

### Faux-amis anglais-français

| Anglais       | ❌ Mauvaise traduction | ✅ Bonne traduction         |
| ------------- | ---------------------- | --------------------------- |
| Actually      | Actuellement           | En fait, en réalité         |
| Eventually    | Éventuellement         | Finalement, à terme         |
| Library       | Librairie              | Bibliothèque                |
| To assist     | Assister               | Aider, accompagner          |
| Comprehensive | Compréhensif           | Exhaustif, complet          |
| To resume     | Résumer                | Reprendre, continuer        |
| Sensitive     | Sensible               | Sensible (correct), délicat |
| Ultimate      | Ultime (= dernier)     | ⚠️ Voir ci-dessous          |

### Le piège « Ultimate »

En anglais, « ultimate » signifie souvent **« le meilleur, le suprême »**.  
En français, « ultime » signifie **« le dernier, le final »**.

```
✅ « Le studio bilingue par excellence »
✅ « Le studio bilingue définitif »
❌ « Le studio bilingue ultime » (sens de "dernier", pas "meilleur")
```

### Autres pièges courants

| Anglais   | Contexte   | Traduction correcte             |
| --------- | ---------- | ------------------------------- |
| Save      | Bouton     | Enregistrer (pas "Sauvegarder") |
| Delete    | Bouton     | Supprimer                       |
| Cancel    | Bouton     | Annuler                         |
| Submit    | Formulaire | Envoyer, Valider                |
| Settings  | Menu       | Paramètres (pas "Réglages")     |
| Dashboard | Interface  | Tableau de bord                 |

## 5. Grammaire & Syntaxe

### Structure de phrase

- **Sujet-Verbe-Complément (SVC)** standard
- Adjectifs généralement **après** le nom : « un livre bilingue »
- Certains adjectifs courts avant : « un beau projet »

### Accents obligatoires

Les accents changent le sens — ne jamais les omettre :

| Correct         | Incorrect        |
| --------------- | ---------------- |
| À (préposition) | A (verbe avoir)  |
| Où (lieu)       | Ou (alternative) |
| Ça (cela)       | Ca (inexistant)  |

### Majuscules

- **Minuscule** pour les langues : « le français », « l'anglais »
- **Majuscule** pour les nationalités : « les Français »
- **Titre de fonctionnalité** : Première lettre majuscule uniquement

```
✅ « Exporter en PDF »
❌ « Exporter En PDF »
```

## 6. Typographie Française

### Espaces insécables

En français, certains signes de ponctuation nécessitent une **espace insécable avant** :

| Signe             | Règle                         | Exemple                 |
| ----------------- | ----------------------------- | ----------------------- |
| : (deux-points)   | Espace insécable avant        | « Titre : description » |
| ; (point-virgule) | Espace insécable avant        | « premier ; deuxième »  |
| ! (exclamation)   | Espace insécable avant        | « Bravo ! »             |
| ? (interrogation) | Espace insécable avant        | « Comment ? »           |
| « » (guillemets)  | Espaces insécables intérieurs | « Texte »               |
| % (pourcentage)   | Espace insécable avant        | « 50 % »                |

### Guillemets

- Utilisez les **guillemets français** : « » (chevrons)
- Pas les guillemets anglais : " "

```
✅ « Votre projet a été enregistré »
❌ "Votre projet a été enregistré"
```

### Nombres

### Dates

| Format        | Exemple          |
| ------------- | ---------------- |
| Date complète | 30 décembre 2025 |
| Date abrégée  | 30/12/2025       |
| Heure         | 14 h 30 ou 14:30 |

## 8. Patterns pour l'Interface

### Boutons

- Verbes à l'**infinitif** ou **impératif** (vouvoiement)
- 1-3 mots maximum

| Action | Français    |
| ------ | ----------- |
| Save   | Enregistrer |
| Export | Exporter    |
| Delete | Supprimer   |
| Cancel | Annuler     |
| Create | Créer       |
| Edit   | Modifier    |

### Messages d'erreur

- Soyez **précis** sur le problème
- Proposez une **solution**
- N'accusez pas l'utilisateur

```
✅ « Connexion perdue. Vérifiez votre connexion internet et réessayez. »
❌ « Erreur réseau »
❌ « Vous avez perdu la connexion »
```

## 10. Considérations Culturelles

### Variantes régionales

| Région       | Particularités                                                             |
| ------------ | -------------------------------------------------------------------------- |
| **France**   | Standard, référence                                                        |
| **Québec**   | Termes anglais traduits plus systématiquement (« courriel » vs « e-mail ») |
| **Belgique** | Quelques variations (« septante » vs « soixante-dix »)                     |
| **Suisse**   | « Natel » pour téléphone portable                                          |

### Pour Synoptic

- Utilisez le **français standard (France)** comme base
- Évitez les anglicismes quand une alternative française existe

```
✅ « Téléverser » (ou « Importer »)
⚠️ « Uploader » (anglicisme toléré mais évitable)
```
