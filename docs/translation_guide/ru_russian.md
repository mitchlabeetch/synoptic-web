## 1. Тон и регистр

### Для Synoptic Studio

- **Регистр**: Профессиональный, но доступный
- **Тон**: Уверенный, ясный и полезный — никогда снисходительный
- **Голос**: Активный залог предпочтителен; пассивный только для акцента на действии

### Ключевые принципы

```
✅ «Сохрани свой проект» (прямой, активный)
❌ «Твой проект будет сохранён» (пассивный)

✅ «Создавай красивые двуязычные книги»
❌ «Двуязычные книги могут быть созданы тобой»
```

## 2. Ты vs. Вы

### Правило для Synoptic

**Используйте «ты»** — это стандарт для современных технологических приложений.

| Контекст                   | Форма  | Пример                           |
| -------------------------- | ------ | -------------------------------- |
| Пользовательский интерфейс | **Ты** | «Сохрани свой проект»            |
| Техническая документация   | **Ты** | «Ты можешь экспортировать в PDF» |
| Сообщения об ошибках       | **Ты** | «Проверь своё подключение»       |
| Юридические тексты         | **Вы** | «Пользователь соглашается...»    |

### Написание местоимений

- **Ты/твой** — со строчной буквы в UI
- **Вы/Ваш** — с заглавной буквы при вежливом обращении к одному человеку

```
✅ UI: « Сохрани свой проект »
✅ Официально: « Сохраните Ваш проект »
```

## 3. Гендерно-нейтральный язык

### Сложности в русском языке

Русский имеет грамматический род (мужской, женский, средний), что усложняет нейтральность.

### Рекомендуемые стратегии

| Метод                           | Пример                 | Применение              |
| ------------------------------- | ---------------------- | ----------------------- |
| **Безличные конструкции**       | «Можно экспортировать» | ✅ Предпочтительно      |
| **Множественное число**         | «Пользователи могут»   | ✅ Нейтрально           |
| **Нейтральные существительные** | «Лицо», «сторона»      | Для юридических текстов |
| **Двойные формы**               | «Авторы и авторки»     | Редко, неформально      |

### Для Synoptic

```
✅ « Добро пожаловать! » (нейтрально)
❌ « Добро пожаловать, дорогой пользователь »

✅ « Для тех, кто создаёт и публикует »
✅ « Для создателей контента »
❌ « Для авторов » (мужской род по умолчанию)

✅ « Твоя команда »
```

### Безличные глаголы

```
✅ « Проект сохранён » (безлично)
✅ « Экспортировать можно в любой формат »
```

## 4. Ложные друзья и ловушки

### Ложные друзья английско-русские

| Английский  | ❌ Неправильный перевод | ✅ Правильный перевод |
| ----------- | ----------------------- | --------------------- |
| Accurate    | Аккуратный              | Точный                |
| Actual      | Актуальный              | Фактический, реальный |
| Brilliant   | Бриллиантовый           | Блестящий, отличный   |
| Cabinet     | Кабинет (комната)       | Шкаф                  |
| Magazine    | Магазин                 | Журнал                |
| Prospect    | Проспект                | Перспектива           |
| Realization | Реализация              | Осознание             |
| Replica     | Реплика                 | Копия, точная копия   |
| Sympathy    | Симпатия                | Сочувствие            |

### Ловушка «Ultimate»

В английском «ultimate» часто означает **«лучший, непревзойдённый»**.  
В русском «ультимативный» связано с **ультиматумом** (требование).

```
✅ « Лучшая двуязычная студия »
✅ « Идеальная двуязычная студия »
❌ « Ультимативная двуязычная студия » (негативная коннотация)
```

## 5. Грамматика и синтаксис

### Структура предложения

- **Подлежащее-Сказуемое-Дополнение (ПСД)** — стандарт, но порядок слов гибкий
- Флексии позволяют менять порядок для акцента

### Падежи

Русский имеет 6 падежей — критически важно для правильности:

| Падеж        | Вопрос        | Пример      |
| ------------ | ------------- | ----------- |
| Именительный | Кто? Что?     | проект      |
| Родительный  | Кого? Чего?   | проекта     |
| Дательный    | Кому? Чему?   | проекту     |
| Винительный  | Кого? Что?    | проект      |
| Творительный | Кем? Чем?     | проектом    |
| Предложный   | О ком? О чём? | (о) проекте |

### Примеры

```
✅ « Сохрани свой проект » (винительный)
✅ « Настройки проекта » (родительный)
✅ « Работай над проектом » (творительный)
```

## 6. Типографика

### Кавычки

- Русские кавычки: **« »** (ёлочки) — основные
- Вложенные: **„ "** (лапки)

```
✅ « Твой проект „Моя книга" сохранён »
```

### Числа и форматы

| Тип        | Формат               | Пример     |
| ---------- | -------------------- | ---------- |
| Тысячи     | Пробел               | 1 000      |
| Десятичные | Запятая              | 3,14       |
| Валюта (€) | X,XX €               | 12,99 €    |
| Валюта (₽) | X,XX ₽ или X,XX руб. | 1 299 ₽    |
| Процент    | X% или X %           | 50%        |
| Дата       | ДД.ММ.ГГГГ           | 30.12.2025 |
| Время      | ЧЧ:ММ                | 14:30      |

### Буква Ё

Рекомендуется использовать **«ё»** во всех случаях для ясности:

```
✅ « Всё готово »
✅ « Твоё имя »
⚠️ « Все готово » (может читаться как «все»)
```

## 7. Паттерны интерфейса

### Кнопки

- Используйте **инфинитив** или **повелительное наклонение** (ты)
- Максимум 1-3 слова

| Действие | Русский                  |
| -------- | ------------------------ |
| Save     | Сохранить                |
| Export   | Экспортировать / Экспорт |
| Delete   | Удалить                  |
| Cancel   | Отмена                   |
| Create   | Создать                  |
| Edit     | Редактировать            |
| Sign in  | Войти                    |
| Sign out | Выйти                    |

### Сообщения об ошибках

- Будьте **конкретны** о проблеме
- Предложите **решение**
- Не вините пользователя

```
✅ « Соединение потеряно. Проверь интернет-подключение и попробуй снова. »
❌ « Ошибка сети »
❌ « Ты потерял соединение »
```

## 9. Культурные аспекты

### Стиль общения

Русские пользователи ценят:

- Ясность и прямоту
- Профессионализм без излишней формальности
- Грамматическую точность

### Англицизмы

Русский принимает многие англицизмы в IT:

- ✅ « Файл », « Папка », « Скачать »
- ⚠️ « Аплоад » → лучше « Загрузить »
- ⚠️ « Линк » → можно « Ссылка »
