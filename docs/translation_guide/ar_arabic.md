## 1. اتجاه النص (RTL)

### قواعد أساسية

- النص العربي يُكتب من **اليمين إلى اليسار**
- الأرقام العربية والإنجليزية تُقرأ من اليسار إلى اليمين
- واجهة المستخدم يجب أن تكون **معكوسة** (mirrored)

### تأثيرات على التصميم

```
✅ الأزرار والقوائم على اليمين
✅ شريط التمرير على اليسار
✅ إشارات الاتجاه معكوسة (→ تصبح ←)
```

### النصوص المختلطة (Bidirectional)

عند دمج العربية مع الإنجليزية أو الأرقام:

```
✅ «تم تصدير الملف إلى PDF بنجاح»
✅ «لديك 5 مشاريع جديدة»
```

## 2. مستوى الخطاب

### لـ Synoptic

**استخدم صيغة المخاطب المحترمة** — المعيار للتطبيقات المهنية.

| السياق         | الصيغة              | المثال                  |
| -------------- | ------------------- | ----------------------- |
| واجهة المستخدم | **أنتَ** (مذكر عام) | «احفظ مشروعك»           |
| رسمي جداً      | **حضرتك**           | «يرجى حفظ مشروعكم»      |
| رسائل الخطأ    | **محترم**           | «يرجى التحقق من اتصالك» |

### ⚠️ ملاحظة مهمة

- صيغة المذكر تُستخدم تقليدياً كصيغة عامة
- الاتجاه نحو استخدام صيغ محايدة يتزايد

## 3. اللغة المحايدة جنسياً

### التحدي

العربية لغة ذات جنس نحوي قوي (مذكر/مؤنث). كل فعل وصفة تتغير حسب الجنس.

### استراتيجيات موصى بها

| الطريقة                 | المثال                            | الاستخدام         |
| ----------------------- | --------------------------------- | ----------------- |
| **صيغة الجمع**          | «المستخدمون والمستخدمات»          | رسمي              |
| **الخطاب العام**        | «عزيزي المستخدم/عزيزتي المستخدمة» | مباشر             |
| **صيغة المبني للمجهول** | «تم الحفظ»                        | ✅ **مفضل للـUI** |
| **الاسم الجمعي**        | «فريق العمل» بدلاً من «العاملون»  | ✅ طبيعي          |

### أمثلة عملية

```
✅ «مرحباً بك!» (محايد بالسياق)
✅ «تم حفظ المشروع بنجاح» (مبني للمجهول)
❌ «مرحباً أيها المستخدم» (مذكر صريح)

✅ «للمبدعين والمبدعات»
✅ «لصانعي المحتوى» (جمع مذكر سالم - يشمل الجنسين تقليدياً)
```

## 4. أخطاء الترجمة الشائعة

### الأصدقاء الزائفون (False Friends)

| الإنجليزية | ❌ ترجمة خاطئة  | ✅ ترجمة صحيحة    | السبب               |
| ---------- | --------------- | ----------------- | ------------------- |
| Ultimate   | الأخير/النهائي  | الأمثل/المثالي    | «ultimate» = الأفضل |
| Save       | أنقذ            | احفظ              | سياق مختلف          |
| Library    | المكتبة (للكتب) | المكتبة (برمجياً) | نفس الكلمة مقبولة   |
| Character  | شخصية           | حرف               | في البرمجيات = حرف  |
| Actually   | فعلياً          | في الحقيقة        | —                   |
| Eventually | حتمياً          | في النهاية        | —                   |

### ترجمة «Ultimate»

في الإنجليزية، «ultimate» تعني **«الأفضل، المثالي»**.

```
✅ «الاستوديو ثنائي اللغة المثالي»
✅ «أفضل استوديو ثنائي اللغة»
❌ «آخر استوديو ثنائي اللغة» (تعني النهاية)
```

## 5. القواعد النحوية

### بنية الجملة

العربية تتبع بشكل رئيسي **VSO** (فعل-فاعل-مفعول) أو **SVO**:

```
✅ «احفظ المشروع» (فعل أمر + مفعول)
✅ «المستخدم يحفظ المشروع» (فاعل + فعل + مفعول)
```

### الإعراب

العربية لغة إعرابية — الحركات تحدد الوظيفة النحوية:

- الفتحة للمفعول به
- الضمة للفاعل
- الكسرة للمجرور

### التشكيل

في الواجهات، عادةً **لا يُستخدم التشكيل** إلا للوضوح:

```
✅ «حفظ» (بدون تشكيل — واضح)
✅ «اِقرأ» (بتشكيل للتوضيح إذا لزم)
```

## 10. الاعتبارات الثقافية

### الاحترام والرسمية

- استخدم صيغ التحية المناسبة
- تجنب الإفراط في المباشرة
- «يرجى» و«شكراً» مهمتان

### الرموز والألوان

- اللون الأخضر إيجابي في الثقافة الإسلامية
- تجنب الرموز الحساسة ثقافياً

### التواريخ

- التقويم الهجري مستخدم في بعض السياقات
- التقويم الميلادي مُستخدم في التطبيقات التقنية
