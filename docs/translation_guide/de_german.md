## 1. Ton & Register

### Für Synoptic Studio

- **Register**: Professionell, aber zugänglich
- **Ton**: Selbstbewusst, klar und hilfsbereit — nie herablassend
- **Stimme**: Aktiv bevorzugt; Passiv nur zur Betonung der Handlung

### Kernprinzipien

```
✅ „Speichern Sie Ihr Projekt" (direkt, aktiv)
❌ „Ihr Projekt wird gespeichert" (passiv)

✅ „Erstellen Sie wunderschöne zweisprachige Bücher"
❌ „Zweisprachige Bücher können von Ihnen erstellt werden"
```

---

## 2. Siezen vs. Duzen

### Regel für Synoptic

**Verwenden Sie das Siezen (Sie)** — es ist der Standard für professionelle Anwendungen.

| Kontext                  | Form           | Beispiel                         |
| ------------------------ | -------------- | -------------------------------- |
| Benutzeroberfläche       | **Sie**        | „Speichern Sie Ihr Projekt"      |
| Technische Dokumentation | **Sie**        | „Sie können als PDF exportieren" |
| Fehlermeldungen          | **Sie**        | „Überprüfen Sie Ihre Verbindung" |
| Lockeres Marketing       | **Du** möglich | „Erstelle dein erstes Buch"      |

### ⚠️ Groß- und Kleinschreibung

- **Sie/Ihr/Ihnen** werden immer **großgeschrieben** (formelle Anrede)
- **du/dein/dir** werden in der neuen Rechtschreibung kleingeschrieben

```
✅ „Haben Sie Ihre Änderungen gespeichert?"
✅ „Hast du deine Änderungen gespeichert?" (informell)
```

## 3. Gendergerechte Sprache

### Empfohlene Optionen

| Methode             | Beispiel           | Verwendung                  |
| ------------------- | ------------------ | --------------------------- |
| **Genderstern**     | Nutzer\*innen      | Empfohlen, sehr verbreitet  |
| **Doppelpunkt**     | Nutzer:innen       | Screenreader-freundlich     |
| **Binnen-I**        | NutzerInnen        | Veraltet, weniger empfohlen |
| **Schrägstrich**    | Nutzer/innen       | Amtlich, aber sperrig       |
| **Neutralisierung** | Nutzende, Personen | Bevorzugt für Fließtext     |

### Empfehlung für Synoptic

```
✅ „Willkommen!" (neutral)
❌ „Willkommen, lieber Benutzer" (männlich)

✅ „Für Autor*innen und Verleger*innen"
✅ „Für alle, die publizieren möchten" (neutral)

✅ „Ihre Anmeldedaten"
❌ „Lieber Kunde/Liebe Kundin" (umständlich)
```

### Substantivierte Partizipien

Eine elegante Lösung für Pluralformen:

| Statt      | Lieber                     |
| ---------- | -------------------------- |
| Benutzer   | Nutzende                   |
| Autoren    | Schreibende / Autor\*innen |
| Übersetzer | Übersetzende               |
| Abonnenten | Abonnierende               |

## 4. Falsche Freunde & Übersetzungsfallen

### Deutsch-Englische Falsche Freunde

| Englisch    | ❌ Falsche Übersetzung | ✅ Richtige Übersetzung     |
| ----------- | ---------------------- | --------------------------- |
| Actually    | Aktuell                | Tatsächlich, eigentlich     |
| Become      | Bekommen               | Werden                      |
| Brand       | Brand                  | Marke                       |
| Gift        | Gift                   | Geschenk                    |
| Handy       | Handy                  | Praktisch, handlich         |
| Recipe      | Rezept (med.)          | Rezept (Kochen) ✓           |
| Sensible    | Sensibel               | Vernünftig, sinnvoll        |
| Sympathetic | Sympathisch            | Mitfühlend, verständnisvoll |

### Die „Ultimate"-Falle

Im Englischen bedeutet „ultimate" oft **„das Beste, das Ultimative"**.  
Im Deutschen bedeutet „ultimativ" eher **„endgültig, letztes Mittel"** mit leicht negativer Konnotation.

```
✅ „Das ultimative zweisprachige Studio" (korrekt im Werbekontext)
✅ „Das beste zweisprachige Studio"
⚠️ „Ultimativ" kann als „endgültig/abschließend" missverstanden werden
```

### Häufige Übersetzungsfehler

| Englisch  | Kontext     | Richtige Übersetzung                         |
| --------- | ----------- | -------------------------------------------- |
| Save      | Button      | Speichern                                    |
| Delete    | Button      | Löschen                                      |
| Cancel    | Button      | Abbrechen                                    |
| Submit    | Formular    | Absenden                                     |
| Settings  | Menü        | Einstellungen                                |
| Dashboard | Interface   | Übersicht / Dashboard                        |
| Privacy   | Datenschutz | Datenschutz (nicht „Privatsphäre" für legal) |

## 5. Grammatik & Satzbau

### Satzstruktur

- Hauptsätze: **Subjekt-Verb-Objekt** (SVO)
- Nebensätze: Verb am **Ende**
- Verbklammer beachten

```
✅ „Sie können Ihr Projekt exportieren."
✅ „Wenn Sie fertig sind, können Sie das Projekt exportieren."
```

### Zusammengesetzte Wörter

Deutsche Komposita werden **zusammengeschrieben**:

```
✅ Projekteinstellungen
❌ Projekt-Einstellungen (nur bei Unklarheit)
❌ Projekt Einstellungen

✅ PDF-Export (Abkürzung + Substantiv)
✅ E-Mail (Bindestrich bei Lehnwörtern)
```

### Groß- und Kleinschreibung

- **Substantive immer groß**: Projekt, Datei, Einstellungen
- **Verben klein**: speichern, exportieren
- **Höflichkeitsform groß**: Sie, Ihr, Ihnen

## 6. Typografie

### Anführungszeichen

- Deutsche Anführungszeichen: **„ "** (unten-oben)
- Alternativ: **» «** (Guillemets, in der Schweiz)

```
✅ „Ihr Projekt wurde gespeichert"
❌ "Ihr Projekt wurde gespeichert" (englisch)
```

### Zahlen und Formate

| Typ       | Format     | Beispiel   |
| --------- | ---------- | ---------- |
| Tausender | Punkt      | 1.000      |
| Dezimal   | Komma      | 3,14       |
| Währung   | X,XX €     | 12,99 €    |
| Prozent   | X %        | 50 %       |
| Datum     | TT.MM.JJJJ | 30.12.2025 |
| Uhrzeit   | HH:MM Uhr  | 14:30 Uhr  |

### Abstände

- **Geschütztes Leerzeichen** vor Einheiten: `50 %`, `12,99 €`
- Kein Leerzeichen bei Bindestrich-Komposita: `E-Mail`

## 7. UI-Muster

### Buttons

- Verwenden Sie den **Infinitiv** oder **Imperativ (Sie-Form)**
- Maximal 1-3 Wörter

| Aktion   | Deutsch     |
| -------- | ----------- |
| Save     | Speichern   |
| Export   | Exportieren |
| Delete   | Löschen     |
| Cancel   | Abbrechen   |
| Create   | Erstellen   |
| Edit     | Bearbeiten  |
| Sign in  | Anmelden    |
| Sign out | Abmelden    |

### Fehlermeldungen

- Seien Sie **genau** über das Problem
- Bieten Sie eine **Lösung** an
- Geben Sie nicht dem Nutzer die Schuld

```
✅ „Verbindung unterbrochen. Überprüfen Sie Ihre Internetverbindung und versuchen Sie es erneut."
❌ „Netzwerkfehler"
❌ „Sie haben die Verbindung verloren"
```

| Januar            | Januar      | Jänner     | Januar               |

- Behalten Sie **ß** bei (außer für Schweizer Lokalisierung)

## 10. Kulturelle Aspekte

### Formalität

Deutsche Nutzer erwarten:

- Höfliche, formelle Ansprache (Sie)
- Klare, präzise Formulierungen
- Sorgfältige Grammatik

### Datenschutz-Sensibilität

Deutsche Nutzer sind besonders sensibel für:

- Datenschutzerklärungen
- DSGVO-Konformität
- Transparenz bei Datennutzung

```
✅ „Ihre Daten werden verschlüsselt gespeichert"
✅ „Wir verwenden Ihre Manuskripte nicht für KI-Training"
```
