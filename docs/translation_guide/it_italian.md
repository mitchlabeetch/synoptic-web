## 1. Tono e Registro

### Per Synoptic Studio

- **Registro**: Professionale ma accessibile
- **Tono**: Sicuro, chiaro e di aiuto — mai condiscendente
- **Voce**: Attiva preferita; passiva solo per enfatizzare l'azione

### Principi chiave

```
✅ «Salva il tuo progetto» (diretto, attivo)
❌ «Il tuo progetto verrà salvato» (passivo)

✅ «Crea splendidi libri bilingui»
❌ «I libri bilingui possono essere creati da te»
```

## 2. Dare del Tu vs. Dare del Lei

### Regola per Synoptic

**Utilizzare il "tu"** — è lo standard moderno per le applicazioni tecnologiche.

| Contesto               | Forma   | Esempio                             |
| ---------------------- | ------- | ----------------------------------- |
| Interfaccia utente     | **Tu**  | «Salva il tuo progetto»             |
| Documentazione tecnica | **Tu**  | «Puoi esportare in PDF»             |
| Messaggi di errore     | **Tu**  | «Controlla la tua connessione»      |
| Termini legali         | **Lei** | «L'utente accetta...» / Impersonale |

### ⚠️ Forme di cortesia

Il "Lei" formale richiede la terza persona singolare:

```
Informale (tu): « Salva il tuo progetto »
Formale (Lei): « Salvi il Suo progetto »
```

### Raccomandazione

Per un'applicazione tech moderna come Synoptic, **il tu è preferibile** per creare un rapporto diretto e amichevole con l'utente.

## 3. Linguaggio Inclusivo

### Opzioni consigliate

| Metodo                    | Esempio                      | Uso                           |
| ------------------------- | ---------------------------- | ----------------------------- |
| **Schwa (ə)**             | utentə                       | Emergente, non standardizzato |
| **Asterisco**             | utent\*                      | Visibile ma non pronunciabile |
| **Riformulazione neutra** | chi utilizza, persona utente | ✅ Preferito                  |
| **Doppia forma**          | utenti e utenti              | Formale ma lungo              |

### Per Synoptic

```
✅ « Benvenuto! » o « Ti diamo il benvenuto! »
❌ « Benvenuto/a »

✅ « Chi scrive e pubblica »
✅ « Le persone che traducono »
❌ « Gli autori »

✅ « Il tuo team »
❌ « I tuoi colleghi »
```

### Nota sulla schwa (ə)

La schwa è sempre più usata in contesti progressivi ma **non è ancora standard**. Per Synoptic, preferire:

- Riformulazioni neutre
- Plurali inclusivi
- Evitare asterischi e schwa nella UI principale

### La trappola «Ultimate»

In inglese, «ultimate» significa spesso **«il migliore, il definitivo»**.  
In italiano, «ultimo» significa **«il finale»**, non necessariamente il migliore.

```
✅ « Lo studio bilingue definitivo »
✅ « Il miglior studio bilingue »
❌ « L'ultimo studio bilingue » (suona come "l'ultimo rimasto")
```

## 5. Grammatica e Sintassi

### Struttura della frase

- **Soggetto-Verbo-Oggetto (SVO)** è standard ma flessibile
- L'italiano permette di **omettere il soggetto** quando è chiaro:
  - ✅ « Salva il tuo progetto » (soggetto implicito: tu)

### Posizione degli aggettivi

- Generalmente **dopo il sostantivo**: « un libro bilingue »
- Alcuni prima con significato diverso:
  - « un grande progetto » (eccellente) vs. « un progetto grande » (dimensioni)

### Articoli

- L'italiano usa articoli più spesso dell'inglese:
  - ✅ « Apri il progetto » (non "Apri progetto")

## 6. Tipografia Italiana

### Virgolette

- Virgolette italiane preferite: **« »** (caporali o sergenti)
- Alternative valide: **" "** (inglesi alte)
- Virgolette singole per citazioni interne: **' '**

```
✅ « Il tuo progetto "Il mio libro" è stato salvato »
```

### Accenti

Gli accenti sono **obbligatori** e cambiano il significato:

| Con accento       | Senza accento    |
| ----------------- | ---------------- |
| È (verbo essere)  | E (congiunzione) |
| Sì (affermazione) | Si (pronome)     |
| Là (avverbio)     | La (articolo)    |
| Perché            | Perche (errato)  |

### Numeri e formati

| Tipo        | Formato    | Esempio    |
| ----------- | ---------- | ---------- |
| Migliaia    | Punto      | 1.000      |
| Decimali    | Virgola    | 3,14       |
| Valuta      | X,XX €     | 12,99 €    |
| Percentuale | X% o X %   | 50%        |
| Data        | GG/MM/AAAA | 30/12/2025 |
| Ora         | HH:MM      | 14:30      |

### Spazi

- **Nessuno spazio** prima dei due punti: « Titolo: descrizione »
- Spazio prima di **%** opzionale: « 50% » o « 50 % »

## 7. Regionalismi

### Italiano standard

L'italiano ha meno variazioni regionali nello scritto rispetto ad altre lingue. Tuttavia:

| Termine | Standard | Regionale             |
| ------- | -------- | --------------------- |
| Anguria | ✅       | Cocomero (Centro-Sud) |
| Ragazzo | ✅       | Giovane (formale)     |

### Per Synoptic

- Usare l'**italiano standard** (basato sul toscano colto)
- Evitare dialettalismi e regionalismi
- Preferire termini universalmente compresi

## 8. Pattern per l'Interfaccia

### Pulsanti

- Usare l'**infinito** o l'**imperativo (tu)**
- Massimo 1-3 parole

| Azione   | Italiano |
| -------- | -------- |
| Save     | Salva    |
| Export   | Esporta  |
| Delete   | Elimina  |
| Cancel   | Annulla  |
| Create   | Crea     |
| Edit     | Modifica |
| Sign in  | Accedi   |
| Sign out | Esci     |

### Messaggi di errore

- Sii **specifico** sul problema
- Offri una **soluzione**
- Non incolpare l'utente

```
✅ « Connessione persa. Controlla la tua connessione internet e riprova. »
❌ « Errore di rete »
❌ « Hai perso la connessione »
```

## 10. Considerazioni Culturali

### Formalità

- L'italiano tech moderno preferisce il tu
- I termini legali possono usare forme impersonali
- Evitare eccessiva formalità («Gentile utente»)

### Anglicismi

L'italiano accetta molti anglicismi nel tech:

- ✅ « Cloud », « download », « upload », « file »
- Ma preferire equivalenti italiani quando esistono e sono naturali:
  - ✅ « Scarica » invece di « Fai il download »
