## 1. 톤과 레지스터

### Synoptic Studio 용

- **레지스터**: 존댓말 (해요체)
- **톤**: 신뢰감 있고, 명확하며, 친절함 — 절대 거만하지 않게
- **문체**: 능동태 선호; 피동태는 행위를 강조할 때만

### 핵심 원칙

```
✅ 「프로젝트를 저장하세요」(직접적, 능동적)
❌ 「프로젝트가 저장됩니다」(수동적)

✅ 「아름다운 이중언어 책을 만들어 보세요」
❌ 「이중언어 책이 당신에 의해 만들어질 수 있습니다」
```

## 2. 존댓말 체계

### Synoptic 규칙

**해요체를 사용** — 현대 비즈니스 앱의 표준입니다.

| 상황        | 스타일               | 예시                            |
| ----------- | -------------------- | ------------------------------- |
| UI 전반     | **해요체**           | 「저장해 주세요」「저장하세요」 |
| 버튼        | **명령형**           | 「저장」「저장하기」            |
| 오류 메시지 | **해요체**           | 「연결을 확인해 주세요」        |
| 마케팅      | **해요체/해체 혼용** | 「지금 시작해 보세요」          |

### 존댓말 수준 비교

| 수준            | 종결어미        | 격식      | UI에서의 사용    |
| --------------- | --------------- | --------- | ---------------- |
| **합쇼체**      | -습니다/-ㅂ니다 | 가장 격식 | 공식 문서        |
| **해요체**      | -요             | 정중+친근 | ✅ **권장**      |
| **해체 (반말)** | -어/-아         | 비격식    | ❌ UI에서 부적절 |

### ⚠️ 주의사항

- 한국어는 높임법이 매우 중요함
- 대상을 낮추는 표현 절대 사용 금지
- 「-해 드리다」로 서비스 정신 표현 가능: 「저장해 드렸습니다」

## 3. 성 중립적 언어

### 한국어의 특징

한국어 명사는 문법적 성별이 없습니다. 이는 자연스럽게 성 중립적입니다.

### 대명사 사용

| 대명사 | 성별      | 권장               |
| ------ | --------- | ------------------ |
| 그     | 남성      | 문맥에 따라        |
| 그녀   | 여성      | UI에서 피함        |
| 그분   | 높임/중립 | ✅ 사용 가능       |
| (생략) | —         | ✅ 가장 자연스러움 |

### 모범 사례

```
✅ 「사용자는 프로젝트를 저장할 수 있습니다」
✅ 「프로젝트를 저장할 수 있습니다」(주어 생략 — 자연스러움)
❌ 「그/그녀의 프로젝트」

✅ 「저자와 편집자 여러분」
✅ 「창작자 여러분」
```

## 4. 번역 함정과 실수

### 가짜 친구 (False Friends)

| 영어       | ❌ 잘못된 번역 | ✅ 올바른 번역 | 설명                 |
| ---------- | -------------- | -------------- | -------------------- |
| Save       | 세이브         | 저장           | 게임 외에선 「저장」 |
| Delete     | 딜리트         | 삭제           | 한국어 권장          |
| Actually   | 실제로         | 사실은         | 문맥 주의            |
| Eventually | 이벤트 관련    | 결국, 마침내   | 완전히 다른 의미     |
| Resume     | 이력서         | 다시 시작하다  | 문맥에 따라          |

### 「Ultimate」의 번역

영어 「ultimate」는 **「최고의, 궁극의」**를 의미합니다.

```
✅ 「궁극의 이중언어 스튜디오」— 올바른 의미
✅ 「최고의 이중언어 스튜디오」— 더 자연스러움
❌ 「마지막 이중언어 스튜디오」— 「끝」을 암시
```

## 5. 문법과 구문

### 문장 구조

한국어는 **SOV (주어-목적어-동사)** 구조:

```
영어: I save the project (SVO)
한국어: 나는 프로젝트를 저장한다 (SOV)
```

### 조사의 중요성

조사(particle)가 문장에서 문법적 역할을 결정합니다:

| 조사    | 기능        | 예시                              |
| ------- | ----------- | --------------------------------- |
| 은/는   | 주제 표시   | 「프로젝트**는** 저장되었습니다」 |
| 이/가   | 주어 표시   | 「오류**가** 발생했습니다」       |
| 을/를   | 목적어 표시 | 「파일**을** 삭제합니다」         |
| 에      | 장소/방향   | 「PDF**에** 내보내기」            |
| 에서    | 장소 (동작) | 「클라우드**에서** 동기화」       |
| 의      | 소유        | 「당신**의** 프로젝트」           |
| 로/으로 | 수단/방향   | 「PDF**로** 저장」                |

### 받침 규칙

조사 선택은 앞 단어의 받침(종성) 유무에 따라 달라집니다:

- 받침 있음: 은, 이, 을
- 받침 없음: 는, 가, 를

## 6. 한글 맞춤법

### 띄어쓰기 주의

- 의존명사 앞에서 띄어쓰기: 「저장할 수 있습니다」
- 보조용언 띄어쓰기: 「저장해 주세요」

### 외래어 표기법

- 영어 장음을 반영하지 않음: 「컴퓨터」(not 컴퓨우터)
- 일관된 표기 유지

## 10. 문화적 고려사항

### 존중의 표현

- 고객을 「고객님」으로 호칭 가능
- 「감사합니다」로 행동에 대한 감사 표현
- 겸손한 표현 선호: 「~해 드리겠습니다」

### 디자인 문화

한국 사용자는:

- 깔끔하고 현대적인 UI 선호
- 높임말에 민감함
- 빠른 피드백 기대
